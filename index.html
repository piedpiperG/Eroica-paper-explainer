讲解主线

用 behavior fingerprint 把 profiling 变成在线诊断信号

EROICA 的核心不是「采更多数据」，而是把每个 worker 上关键函数的运行时行为压缩为 beta / mu / sigma 三维指纹，再进行跨 worker 比较，定位异常函数、异常机器和异常资源。

为什么值得讲

论文把一个经典矛盾拆开了：监控看得广，profiling 看得深，但生产环境两者都要。

问题

大规模训练里，用户代码、数据加载、框架、通信、GPU、NIC 和远端存储都可能拖慢全局任务。

矛盾

在线监控低开销、全覆盖，但粗粒度；离线 profiling 细粒度，但数据量巨大，只能抽样。

解法

只在变慢时短时触发 profiling，把 critical path 函数行为压缩成指纹，在中心侧比较定位。
